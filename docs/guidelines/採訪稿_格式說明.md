# 採訪稿格式說明

## 文件用途

採訪稿（Interview Article）是**最終成品文稿**，將直接用於：

- 轉換成 Word 文件（`.docx`）
- 發表於專刊、期刊、網站等平台
- 提供給讀者閱讀的完整文章

## 基本規格

- **檔案格式**：Markdown (`.md`)
- **編碼**：UTF-8
- **命名**：`採訪稿.md`
- **存放位置**：`outputs/採訪稿/`

## 文件結構

### YAML Front Matter（必須）

```yaml
---
type: "採訪稿"
version: "v1"
document_title: "文章標題"
subject_person: "受訪者姓名"
subject_role: "受訪者主要職稱"
source_script: "../腳本/腳本.md"  # 若有對應腳本
source: "原始文稿檔名"
---
```

### 主要章節（必須）

#### 1. 主標

```markdown
# 主標

[文章主標題，約 15-30 字]
```

**說明**：
- 應清楚帶出主題與價值
- 簡潔有力，吸引讀者
- 建議 15-30 字之間

**範例**：
```markdown
# 主標

建立臺灣應變自我品牌，鼎足世界一同守護未來
```

#### 2. 眉標（座右銘）

```markdown
# 眉標（座右銘）

[補充主標的精神，或引用受訪者的關鍵一句話]
```

**說明**：
- 可以是受訪者的座右銘
- 可以是關鍵引言
- 可以是副標題說明
- 通常 30-80 字

**範例**：
```markdown
# 眉標（座右銘）

我不能祈求老天爺的幫助，把弟兄安全帶回是我的責任
能夠讓每個人安全，能夠避免人員跟環境的污染，那才是最完整的應變
```

#### 3. 受訪者經歷

```markdown
# 受訪者經歷

- 現職：
  - [職稱一]
  - [職稱二]
- 重要經歷：
  - [經歷一]
  - [經歷二]
  - [經歷三]
```

**說明**：
- 使用條列式呈現
- 現職放在最前面
- 重要經歷依時間或重要性排序
- 聚焦與文章主題相關的經歷

**範例**：
```markdown
# 受訪者經歷

- 現職：
  - 中華民國化學應變協會（CERA）理事長
  - 財團法人工業技術研究院化災應變研究室 技術經理
  - 環境部環境事故應變諮詢中心 資深諮詢員
- 重要經歷：
  - 曾任新北市政府消防局 化災搶救隊員
  - 參與多起重大化災事故應變
  - 推動臺灣化學應變體系建置
```

#### 4. 摘要

```markdown
# 摘要

[用 3–5 行，濃縮整篇故事的精神與核心價值]
```

**說明**：
- 100-200 字之間
- 概括全文重點
- 可以引發讀者繼續閱讀的興趣
- 涵蓋主要論點與價值

**範例**：
```markdown
# 摘要

雖然臺灣應變體系起步較晚，但在政府與民間共同努力下，已建立起具有特色的化學應變機制。中華民國化學應變協會理事長何大成，從消防員到應變專家，見證並參與了臺灣化災應變的成長歷程，他相信只要持續培育人才、落實企業責任，臺灣的應變能力將能與國際接軌，共同守護未來。
```

#### 5. 內文段落

```markdown
# 段一　[段落標題]

[段落內容，完整的文章段落]

# 段二　[段落標題]

[段落內容]

# 段三　[段落標題]

[段落內容]

# 段四　[段落標題]

[段落內容]
```

**說明**：
- 段落標題使用全形空格分隔（段一　標題）
- 每段應有明確的主題
- 段落之間應有邏輯連貫性
- 段落數量通常 3-5 段，可依實際需求調整
- 保持適度的口語感，但句子要完整通順

**段落編號格式**：
- 段一、段二、段三、段四（中文數字）
- 使用全形空格（　）分隔編號與標題

#### 6. 結語（可選）

```markdown
# 結語

[編輯部總結或展望]
```

**說明**：
- 非必要欄位
- 可用於編輯部的總結
- 或對未來的展望

## 語氣與風格

### 專刊報導風格

- **專業但不艱澀**：使用專業術語但要有說明
- **保留適度口語感**：可使用引言增加真實感
- **句子完整通順**：避免破碎或過於簡略
- **邏輯清晰**：段落之間有明確的起承轉合

### 引言使用

```markdown
何大成理事長表示：「我不能祈求老天爺的幫助，把弟兄安全帶回是我的責任。」
```

- 使用中文引號「」
- 引言應準確呈現受訪者原意
- 適度使用，不要整段都是引言

### 專有名詞

以下專有名詞應保持一致：

- **組織**：CERA、ERIC、化學署、環境部
- **專業術語**：毒化災、化災應變、應變體系
- **人名**：保持全名或約定俗成的稱呼

## 產出要求

1. **完整性**：
   - 所有必要欄位都應填寫
   - 若缺資料則標註 `TODO：[說明]`

2. **一致性**：
   - 專有名詞拼法統一
   - 標點符號統一（使用中文標點）
   - 格式統一

3. **可讀性**：
   - 段落分明
   - 邏輯清晰
   - 語句流暢

4. **準確性**：
   - 人名、職稱正確
   - 數據、日期準確
   - 引言真實

## Markdown 格式規範

### 標題層級

```markdown
# 一級標題（用於章節）
## 二級標題（不建議在採訪稿使用）
```

- 採訪稿只使用一級標題（#）
- 標題與內文之間空一行

### 段落與換行

```markdown
這是第一段。

這是第二段。
```

- 段落之間空一行
- 同段落內要換行，行尾加兩個空格

### 列表

```markdown
- 項目一
- 項目二
  - 子項目
```

### 強調

```markdown
**粗體**（謹慎使用）
*斜體*（中文不建議使用）
```

## 轉換為 Word 後的處理

採訪稿會經由 `scripts/generate_doc_outputs.py` 轉換為：

1. **Word 文件**（`採訪稿_templete.docx`）
   - 套用 `reference.docx` 樣式
   - 保持段落結構

2. **欄位說明 JSON**（`採訪稿說明.json`）
   - 提取關鍵欄位資訊
   - 方便系統追蹤與管理

## 常見問題

### Q: 段落數量一定要四段嗎？
A: 不一定。通常 3-5 段，依內容豐富度調整。

### Q: 一定要有摘要嗎？
A: 是的，摘要是必要欄位。

### Q: 可以使用圖片嗎？
A: Markdown 可以標註圖片位置，但實際插圖建議在 Word 階段處理。

### Q: 段落標題可以自訂嗎？
A: 可以，但應遵循腳本規劃的結構。
