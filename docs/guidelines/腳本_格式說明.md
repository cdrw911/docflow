# 腳本格式說明

## 文件用途

腳本（Script）是用於**內部規劃與採訪設計**的文件，主要目的是：

- 明確文章的核心訊息與結構
- 規劃每個段落的目的與重點
- 作為產出「採訪稿」的藍圖與參考依據

## 基本規格

- **檔案格式**：Markdown (`.md`)
- **編碼**：UTF-8
- **命名**：`腳本.md`
- **存放位置**：`outputs/腳本/`

## 文件結構

### YAML Front Matter（必須）

```yaml
---
type: "腳本"
version: "v1"
document_title: "文件標題"
source: "來源文稿檔名"
---
```

### 主要章節（必須）

#### 1. 基本資訊

```markdown
# 1. 基本資訊

- 任務代碼：[專案編號或代號]
- 刊物名稱：[例如：化學署專刊]
- 目標讀者：[例如：政府機關、企業、專業人士]
- 文體風格：[例如：專訪報導 / 專題故事 / 社論]
```

**說明**：
- 任務代碼：方便追蹤與管理
- 刊物名稱：明確這篇文稿的發表場域
- 目標讀者：幫助確認語氣與深度
- 文體風格：定義寫作方式

#### 2. 受訪者資訊

```markdown
# 2. 受訪者資訊

- 姓名：[全名]
- 職稱與單位：[現職]
- 關鍵身分標籤：[例：毒化災應變專家、CERA 理事長]
- 精簡經歷摘要（3–5 行）：
  [條列或段落形式的重要經歷]
```

**說明**：
- 提供受訪者的核心身分資訊
- 經歷摘要應聚焦在與本文主題相關的背景

#### 3. 主標與眉標

```markdown
# 3. 主標與眉標

- 主標：[文章主標題，約 15-30 字]
- 眉標（或座右銘）：[補充主標的精神，可引用受訪者關鍵句]
- 備選主標（選填）：[其他候選標題]
- 備選眉標（選填）：[其他候選眉標]
```

**說明**：
- 主標：應清楚帶出主題與價值
- 眉標：可以是座右銘、關鍵引言，或補充說明

#### 4. 核心訊息（Key Messages）

```markdown
# 4. 核心訊息（Key Messages）

- 核心訊息 1：[此文章最想傳達的第一個重點]
- 核心訊息 2：[第二個重點]
- 核心訊息 3：[第三個重點]
```

**說明**：
- 列出 2-4 個核心訊息
- 這些訊息應貫穿全文
- 每個段落應至少呼應一個核心訊息

#### 5. 內容結構大綱

```markdown
# 5. 內容結構大綱

## 段一：[段落標題]

- 段落目的：[這一段要達成什麼目標]
- 要帶出的故事／案例：[具體例子或故事]
- 必須出現的關鍵句：[重要引言或論點]

## 段二：[段落標題]

- 段落目的：
- 重點說明：

## 段三：[段落標題]

- 段落目的：
- 重點說明：

## 段四：[段落標題]

- 段落目的：
- 重點說明：
```

**說明**：
- 段落數量可依實際需求調整（通常 3-5 段）
- 每段都應有明確的目的
- 關鍵句可直接引用來源文稿

#### 6. 版面與語氣指引

```markdown
# 6. 版面與語氣指引

- 語氣設定：[例如：專業但親切 / 嚴謹正式 / 故事敘事]
- 行文節奏：[例如：平穩敘事 / 起承轉合 / 問題導向]
- 是否保留第一人稱／引言口語感：[是/否，及說明]
```

**說明**：
- 為後續撰寫採訪稿提供風格指引

#### 7. 校對與禁忌

```markdown
# 7. 校對與禁忌

- 專有名詞列表：[列出需要特別注意的專有名詞及正確寫法]
- 必須確認的數據／日期：[需要驗證的關鍵資訊]
- 禁用用語：[應避免使用的詞彙或表達方式]
```

**說明**：
- 專有名詞：例如 CERA、ERIC、化學署等
- 數據／日期：需要二次確認的資訊
- 禁用用語：可能引起爭議或不恰當的用語

## 產出要求

1. **完整性**：所有欄位都應填寫，若無資料則標註 `TODO：[說明]`
2. **一致性**：專有名詞、格式、標點符號應統一
3. **可讀性**：條列清晰，邏輯分明
4. **可追溯性**：在 front matter 中註明來源文稿

## 注意事項

- 腳本是**規劃文件**，不是最終文稿
- 重點在於**結構與重點規劃**，而非完整文句
- 可以使用條列式、關鍵句方式呈現
- 應為後續「採訪稿」提供明確的寫作方向

- [ ] 專有名詞列表完整
